## 演示案例1：小程序 iOS 底部安全区遮挡（BUG类，模板A）

### 第一步：问题发生（工作现场）

开发小程序底部按钮，iPhone全面屏机型按钮被系统手势栏遮挡，安卓正常。

### 第二步：当场解决问题，记录临时信息

随手记下：机型、现象、临时代码。

### 第三步：下班后/空闲时间，整理成标准笔记（套用模板A）

````markdown
# 小程序 iOS 底部安全区遮挡底部操作按钮

## 1. 问题现象 & 复现环境

1. 现象：iPhone X及以上全面屏手机，页面底部按钮、Tab栏、弹窗底部按钮被系统手势条遮挡；安卓手机无异常。
2. 复现步骤：打开小程序，进入含底部按钮的页面，使用全面屏iPhone查看。
3. 运行环境：
   - 技术栈：MPX 微信小程序 + Vant Weapp
   - 设备：iPhone 12/13/14，iOS 15+
   - 基础库：微信小程序基础库 2.20+

## 2. 问题根因分析

iOS全面屏取消实体Home键，底部存在系统安全区域（手势操作区）。
小程序提供 `safe-area-inset-bottom` 环境变量标识安全区高度，未给底部容器预留内边距，导致元素落入安全区被遮挡。

## 3. 解决方案

### 方案一：标准根治方案（全局通用，推荐）

使用小程序内置安全区环境变量，全局统一设置底部内边距，一套样式兼容双端。

### 方案二：临时规避方案（不推荐，硬写固定高度）

直接给底部容器设置固定 `padding-bottom: 34rpx`，不同机型适配差，仅应急使用。

## 4. 完整代码示例

### 1）全局页面通用适配（所有页面生效）

```css
/* app.wxss / 全局公共样式 */
page {
  /* 兼容旧版iOS */
  padding-bottom: constant(safe-area-inset-bottom);
  /* 标准写法 */
  padding-bottom: env(safe-area-inset-bottom);
}
```
````

### 2）底部 Tab 栏 / 弹窗单独加强适配

```css
/* 底部导航栏、弹窗底部按钮容器 */
.tab-bar,
.dialog-footer {
  padding-bottom: calc(16rpx + env(safe-area-inset-bottom));
}
```

## 5. 注意事项 & 踩坑补充

- 安卓手机 safe-area-inset-bottom 默认为 0，不会产生多余空白，无需额外判断机型；
- 小程序原生 map、video 等原生组件层级特殊，全屏浮层必须单独加样式；
- 不要单独使用 constant()，低版本 iOS 必须双写法兼容。

## 6. 关联知识点

- 小程序 rpx 自适应单位；
- 小程序全局样式、页面样式优先级；
- 移动端各类安全区适配（H5 / App）。
