# 整套复盘和沉淀流程分为五步：

## 1、即时记录：问题原始信息留存

问题解决第一时间，先记录基础信息，避免后续遗忘细节。主要内容包括：问题现象（界面报错、功能异常、卡顿、兼容问题等）、复现步骤、运行环境（设备、系统、框架版本、项目场景）、初步报错日志、相关代码片段。比如小程序 Canvas 海报模糊、大屏重渲染卡顿、html2canvas 样式丢失这类问题，都要完整记录现场信息

## 2、过程复盘：梳理排查与解决思路

完整梳理从发现问题、分析猜测、逐步排查、尝试方案到最终解决的全流程。重点标注：最初的错误判断、走的弯路、关键排查节点、用到的调试手段（断点、日志、网络面板、真机调试等）。同时区分临时规避方案和根治方案，优先记录最优解，分析问题产生的底层原因，不只是记录 “怎么修好的”，更要搞懂 “为什么会出现这个问题”。

## 3、方案提炼：封装为标准化 Skill 知识点

这是核心环节，把零散的解决方法提炼成结构化的 Skill 存入个人知识库，按照技术分类tag + 问题标题 + 场景 + 解决方案 + 代码示例 + 注意事项统一格式整理。
分类维度会按照技术栈划分：Vue/React、小程序、数据可视化、工程化、网络、兼容性等，方便检索。例如把「小程序 iOS 安全区适配」「html2canvas 跨域与样式问题」「WebSocket 大屏防重渲染」分别建立独立 Skill 卡片，附上可直接复用的代码片段、配置代码、参数配置。

## 4、拓展延伸：举一反三，补充关联知识点

针对当前问题做横向拓展，关联同类场景、相似问题、边界情况。比如解决小程序图片跨域后，会同步补充小程序下载文件、地图接口、网络请求的通用跨域规则；解决前端重排重绘问题后，延伸记录常见优化手段、禁用重排的 CSS 属性。同时标注该 Skill 适用的项目场景、技术栈限制，明确哪些场景可以直接复用，哪些场景需要微调。

## 5、落地复用与定期回顾

一是开发中直接复用：后续新项目、同类问题出现时，直接检索知识库中的 Skill，照搬成熟方案，重复踩坑，提升开发效率；二是定期回顾梳理，每周 / 每月抽时间翻阅知识库，把零散的 Skill 串联成知识体系，比如将所有小程序相关问题整合，总结出一套完整的小程序开发避坑规范；三是同步优化项目，针对复盘发现的共性问题，抽空对历史项目做批量优化，从整体上提升项目质量。

# 具体实操

## 1、统一目录分类

按照技术大模块 → 子分类搭建文件夹 / 目录 转换成tag

## 2、标准化笔记模版

模板 A：问题 / BUG / 踩坑类（技术错题集主力）
模板 B：可复用方案 / 功能实现类（Skill 方案库）

## 3. 命名规则（搜索一秒定位）

标题统一格式：`【技术分类】+ 问题/功能 + 核心关键词`
示例：

- 【小程序】iOS安全区遮挡底部按钮
- 【Vue3】Axios二次封装 & 全局错误拦截
- 【可视化】ECharts 窗口resize自适应

## 4. 增量更新规则

后续你再次遇到**同类问题**：

1. 先去知识库搜索旧笔记；
2. 如果旧方案可用 → 直接复用，无需新增；
3. 如果旧方案有缺陷、新场景有变化 → **在原笔记下方追加「补充内容」**，不新建笔记。
